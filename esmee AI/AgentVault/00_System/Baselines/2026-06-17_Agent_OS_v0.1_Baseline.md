# Agent OS v0.1 Baseline

本文件冻结 esmee AI AgentVault 在 2026-06-17 的 v0.1 可用基线。它记录当前已经可用的能力、明确不能做的边界、已验证状态和下一步建议。

## 1. 基线结论

v0.1 已经可作为本地 Obsidian 行动记忆系统使用。

当前重点不是继续扩展功能，而是用真实任务验证这条最小链路：

1. 判断主责部门和协作部门。
2. 使用 `$obsidian-memory` 或 writer 创建行动记忆。
3. 执行一个小步行动。
4. 使用 `append-log` 记录结果。
5. 使用 `set-status` 收尾。
6. 给用户简短中文汇报。

## 3. 当前可用能力

Obsidian 行动记忆写入器已经支持：

- `create`：创建标准行动记忆 Markdown 文件。
- `append-log`：向已存在的行动记忆追加带时间戳的日志。
- `set-status`：更新 frontmatter 中的 `status` 和 `current_mode`。
- `uri`：生成 `obsidian://open` 链接。
- vault 内路径限制：拒绝逃逸到 vault 外部路径。

全局 `$obsidian-memory` skill 已经支持：

- 任务前判断部门和协作关系。
- 行动前创建或更新行动记忆。
- 失败、阻塞、待确认、跨部门交接的记录规则。
- 高风险状态使用 `坎-艮` 或 `坎-艮-兑`。
- 后续 `append-log`、`set-status`、`uri` 必须复用 `create` 输出的真实路径。

## 4. 明确不能做什么

v0.1 不包含以下能力：

- 不安装 Obsidian 插件。
- 不调用 Obsidian REST API。
- 不修改 `.obsidian` 配置。
- 不自动提交 Git、不创建 Git tag。
- 不自动创建、关闭、归档多个 Codex 线程。
- 不在未经确认时写入 vault 外部目录。
- 不在未经确认时做 GUI 自动化、截图、鼠标键盘控制。
- 不删除、覆盖、批量迁移或外发任何资料。

- 视需要创建 `agent-os-v0.1` tag。

## 7. 下一步建议

推荐下一步执行真实端到端验收任务：

> 用全局 `$obsidian-memory` 处理一个很小的真实任务，从创建行动记忆到最终 `completed` 全链路跑完。

成功标准：

- 生成一条新的行动记忆。
- 行动记忆包含部门判断、资料、风险、禁止事项、执行结果和下一步。
- 至少追加一条执行日志。
- 最终 status 为 `completed` 或合理的 `failed/blocked`。
- 用户能通过 Obsidian URI 打开对应记忆。

## 8. 发布说明

v0.1 的定位是最小稳定内核：

- 先保证记忆写入稳定。
- 再保证失败和权限边界可审计。
- 最后再扩展多部门调度、GUI 自动化或 Git 发布流程。
