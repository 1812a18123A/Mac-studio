# 中文优先检索规范

## 1. 目标

从现在开始，记忆库里的行动记忆、部门交接、复审结果、计划摘要和技能记忆都要以中文为主来检索。

英文可以保留为技术 ID、命令名、仓库名、文件名、API 名或原文引用，但不能取代中文标题、中文摘要和中文关键词。

## 2. 默认规则

新建或更新记忆时，必须优先写入：

- 中文标题。
- 中文摘要。
- 中文关键词。
- 中文别名。
- 中文目录路径。
- 中文文件元素。
- 中文时间元素。
- 中文下一步。
- 中文禁止项。

如果文件名或 `task_id` 使用英文，需要在正文前部补中文检索入口。

## 3. 推荐字段

新记忆的 frontmatter 优先包含：

```text
chinese_title:
中文标题:
aliases:
  - 中文别名 1
  - 中文别名 2
search_keywords:
  - 中文关键词 1
  - 中文关键词 2
检索元素:
  - 中文关键词 1
  - 中文关键词 2
summary_zh:
中文摘要:
directory_zh:
目录中文路径:
file_time_zh:
文件时间中文:
年份中文:
月份中文:
日期中文:
时段中文:
status_zh:
状态中文:
risk_level_zh:
风险中文:
```

字段含义：

- `chinese_title`：中文主标题，用于 Obsidian 搜索和人工浏览。
- `aliases`：用户可能会搜索的中文叫法。
- `search_keywords`：任务、部门、对象、风险、下一步的中文关键词。
- `summary_zh`：一句话中文摘要。
- `目录中文路径`：把英文目录含义转成中文，例如 `记忆库 / 行动日志 / 2026年06月17日`。
- `文件时间中文`：把文件时间转成中文，例如 `2026年06月17日 22点48分`。
- `检索元素`：汇总可搜索的中文部门、对象、状态、风险、时间段和目录名。

## 4. 目录、文件元素和时间

历史目录可以继续保持稳定英文路径，避免破坏 Obsidian 链接；但每个新文件必须写入中文目录解释。

推荐对应关系：

- `AgentVault`：智能体资料库。
- `50_Memory`：记忆库。
- `Action_Logs`：行动日志。
- `Department_Handoffs`：部门交接。
- `60_Skills`：技能库。
- `20_Engineering`：工程资料。

每个新行动记忆必须能用以下中文元素检索：

- 目录元素：记忆库、行动日志、日期。
- 文件元素：行动记忆、主责部门、任务名称、任务状态、风险等级。
- 时间元素：年份、月份、日期、几点几分、上午/下午/晚上等时段。
- 关系元素：相关部门、相关技能、相关文件、下一步。

## 5. 正文要求

正文前 20 行内应能看到：

- 这条记忆是什么。
- 为什么创建。
- 当前状态。
- 中文下一步。
- 相关部门。
- 中文目录。
- 中文时间。
- 中文文件元素。

对跨部门任务，正文中必须保留中文部门名，例如：

- 总控办公室
- 记忆部
- 工程部
- 安全部
- GUI 自动化部
- 技能优化部

## 6. 历史文件策略

不要批量改名历史文件。

原因：

- 避免破坏 Obsidian 链接。
- 避免让 Git 历史出现大量无意义 rename。
- 避免影响已推送的部门交接引用。

历史文件只在再次触达时补中文检索字段或中文摘要。

## 7. 英文保留规则

以下内容可以保留英文：

- Git commit hash。
- GitHub 仓库名。
- CLI 命令名。
- API、插件、skill 名。
- 稳定 task id。
- 原始错误信息或外部项目原文。

但同一段附近应补中文解释，便于中文检索。

## 8. 检查清单

每次写入记忆后自检：

- [ ] 是否有中文标题。
- [ ] 是否有中文摘要。
- [ ] 是否有中文关键词或别名。
- [ ] 是否有中文目录路径。
- [ ] 是否有中文文件元素。
- [ ] 是否有中文时间元素。
- [ ] 是否有中文下一步。
- [ ] 英文 ID 是否有中文解释。
- [ ] 是否避免批量改名历史文件。
