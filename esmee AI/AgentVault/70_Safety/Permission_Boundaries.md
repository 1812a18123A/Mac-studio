# Permission Boundaries

本文件是 esmee AI 多部门协作的基础权限边界。所有部门在执行任务前，应先判断动作是否属于低风险可执行、需要用户确认、或禁止动作。

## 1. 需要用户确认的动作

以下动作必须先暂停并获得用户明确确认：

- 删除、覆盖、批量移动、批量重命名文件或目录。
- 修改 `.obsidian` 配置、Obsidian 插件、vault 结构性设置。
- 修改全局 skill、全局插件、系统配置、shell profile、launch agent、定时任务。
- 安装依赖、安装应用、升级工具、运行会修改系统环境的命令。
- 发送邮件、消息、表单、评论、PR、支付、下单、订阅或任何外部提交。
- 上传本地文件、截图、音频、记忆文件、代码、密钥、日志到外部服务。
- 使用外部网络做检索、下载、API 调用，且会影响决策、成本、隐私或账号状态。
- 启动 GUI 自动化、鼠标键盘控制、窗口操作、屏幕录制或截图采集。
- 写入 vault 外部路径、跨项目写入、全局目录写入。
- 处理密钥、令牌、账号、支付、身份、隐私资料、客户资料或敏感日志。

## 2. 可直接执行的低风险动作

以下动作通常可直接执行，但仍需写入行动记忆：

- 读取用户明确指定的本地文件。
- 在当前 vault 内创建或更新行动记忆。
- 在用户明确指定的 vault 路径内创建单个低风险 Markdown 文档。
- 使用 `rg`、`ls`、`cat`、`sed` 等只读命令收集上下文。
- 运行本地只读检查或已有测试，前提是不安装新依赖、不访问外网、不写入系统目录。
- 生成部门交接包或在行动记忆中追加执行结果。
- 汇总已读取资料、风险、限制、下一步建议。

## 3. 禁止动作

以下动作不得执行，除非用户之后给出新的、明确的授权并改变任务边界：

- 未经确认删除文件、清空目录、执行破坏性 git 操作。
- 未经确认外发、上传、付款、提交表单、发送邮件或消息。
- 绕过系统权限、绕过审批、隐藏执行日志。
- 把本地私密资料、密钥、截图、音频、记忆文件发往外部服务。
- 修改 `.obsidian`、全局 skill、系统配置或插件来规避限制。
- 在没有行动记忆的情况下执行实质性修改。
- 创建多个规范文件来扩大原任务范围。
- 把不确定事项当作确定事实继续执行。

## 4. 多部门交接安全要求

跨部门任务必须使用 Obsidian 交接包或行动记忆记录：

- 发起部门、接收部门、任务 ID、优先级。
- 相关行动记忆、相关 skill、相关插件。
- 当前状态、背景、已完成工作。
- 请求下一部门执行的最小动作。
- 明确禁止事项和需要用户确认的动作。
- 成功标准和结果写入位置。

接收部门不得扩大任务范围。若发现交接包缺少权限、路径或成功标准，应转入 `艮` 模式并请求补充。

## 5. GUI 自动化启用前条件

启动 GUI 自动化前必须满足：

- 用户明确要求或确认可以操作 GUI。
- 已说明目标 App、目标窗口、预期动作、可接受失败范围。
- 已确认不会发送、删除、支付、提交表单或外发资料。
- 已创建行动记忆，并记录当前卦态为 `离-坎` 或包含 `坎`。
- 若需要截图、屏幕读取、鼠标点击、键盘输入，必须记录资料用途和保存位置。
- 高风险界面动作前必须停下，让用户确认。

## 6. Obsidian / 文件写入边界

默认允许的写入范围：

- `AgentVault/50_Memory/Action_Logs/`
- 用户明确指定的 vault 内文件。
- 当前任务明确要求的单一目标文件。

默认需要确认的写入范围：

- `.obsidian/`
- vault 外部路径。
- 全局 skill 目录。
- 插件目录、系统配置目录、shell 配置文件。
- 批量生成、批量迁移、批量重命名文件。

写入前必须先记录：

- 主责部门和协作部门。
- 要改什么。
- 不能做什么。
- 需要哪些资料。
- 是否需要用户确认。
- 风险等级和缓解措施。

## 7. 失败或不确定时的处理方式

遇到失败或不确定时：

- 立即停止扩大执行范围。
- 在行动记忆中记录实际做了什么、失败原因、已知信息、未知信息。
- 标记需要用户确认的事项。
- 如果涉及权限或安全风险，通知安全部。
- 如果涉及记忆结构或行动记录缺失，通知记忆部。
- 如果涉及代码、测试、依赖或实现路径，通知工程部。
- 如果涉及 skill 规则不足或反复失败，通知技能优化部。

## 8. 快速判断

- 能读不写：通常低风险。
- 写入行动记忆：通常低风险。
- 写入用户指定单个 vault 文件：低到中风险，需确认任务边界。
- 删除、外发、上传、支付、安装、全局写入、GUI 控制：高风险，必须确认。
- 不确定：停止、记录、确认。
